Explicación

Este ejercicio muestra un problema de coerción de tipos de variable en Javascript.

Se ilustran algunos de los comportamientos contraintuitivos y peligrosos del operador de igualdad débil (==), razón por la cual se recomienda el uso de operador de igualdad estricta (===) que no realiza coerción de tipos y compara tanto el valor como el tipo de dato. Esto evita bugs difíciles de detectar y hace el código más predecible.
